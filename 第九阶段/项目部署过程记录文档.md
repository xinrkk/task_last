## 一、部署架构概述

生产环境部署采用以下架构：

```
用户请求 → Nginx（反向代理） → Gin后端服务（:8080）
```

Nginx作为反向代理，负责接收外部请求并转发给Gin服务，同时可提供SSL终止、负载均衡、静态文件处理等功能。

## 二、部署步骤

### 1. 服务器环境准备


```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装Nginx
sudo apt install nginx -y

# 安装Go环境
wget https://golang.org/dl/go1.21.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.21.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc
```

**CentOS/RHEL系统：**

```bash
# 安装EPEL仓库
sudo yum install epel-release -y

# 安装Nginx
sudo yum install nginx -y

# 安装Go环境
wget https://golang.org/dl/go1.21.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.21.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc
```

### 2. 上传项目代码到服务器

有多种方式上传代码：

**方式一：使用SCP命令（本地执行）**

```bash
scp -r /本地项目路径/member-system root@你的服务器IP:/root/
```

**方式二：使用Git克隆**

```bash
git clone https://github.com/你的仓库/member-system.git
cd member-system
```

### 3. 编译Go应用

在生产环境，建议编译为静态二进制文件：

```bash
cd /root/member-system

# 下载依赖
go mod download

# 编译为Linux可执行文件
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -o member-server main.go

# 测试运行（Ctrl+C停止）
./member-server
```



### 4. 配置Nginx反向代理

![img](https://raw.githubusercontent.com/dunwu/images/master/cs/web/nginx/reverse-proxy.png)


创建Nginx配置文件：

```bash
sudo nano /etc/nginx/sites-available/member-api
```

**基础HTTP配置：**

```nginx
server {
    listen 80;
    server_name your_domain.com;  # 替换为你的域名或服务器IP

    # 日志配置
    access_log /var/log/nginx/member-access.log;
    error_log /var/log/nginx/member-error.log;

    location / {
        # 转发到Gin服务
        proxy_pass http://127.0.0.1:8080;
        
        # 传递真实客户端信息
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket支持（如需要）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

启用配置：

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/member-api /etc/nginx/sites-enabled/

# 测试配置语法
sudo nginx -t

# 重新加载Nginx
sudo systemctl reload nginx
```

### 5. 配置防火墙

**Ubuntu（使用UFW）：**

```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp  # HTTPS时使用
sudo ufw reload
```

**CentOS（使用firewalld）：**

```bash
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

## 三、测试部署

### 1. 测试服务运行状态

```bash
# 检查Gin服务
curl http://127.0.0.1:8080/ping  # 如果有ping路由

# 检查Nginx转发
curl http://你的服务器IP/register
```

### 2. 测试API接口

使用curl测试登录接口：

```bash
curl -X POST http://你的服务器IP/login \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "admin",
    "password": "admin123"
  }'
```
